# jvot3

Tools for creating and decomposing Lojban lujvo.

jvot3 is the successor to [latkerlo-jvotci](https://github.com/latkerlo/latkerlo-jvotci).
It's a separate crate because, for the most part, jvot3 is a complete rewrite not authored by latkerlo.
jvot3 does take these elements from latkerlo-jvotci:

- lists of consonant clusters, rafsi, etc. (these are just "facts about Lojban" that anyone implementing a jvotci
  would need to have)
- the tests, which are 3 `*.tsv` files: one is the list of every lujvo generated by noralujv, the others are various
  (formerly-)broken inputs we collected over the years
- the modified lujvo scoring algorithm that doesn't reward apostrophes

latkerlo-jvotci is itself based on

- [p-lujvo](https://codeberg.org/tb148/p-lujvo) by Toni Brown
- [jvozba](https://github.com/uakci/jvozba/tree/v3) by uakci
- [sozysozbot jvozba](https://github.com/sozysozbot/sozysozbot_jvozba) by sozysozbot

## Differences from latkerlo-jvotci

jvot3

- is only written in 1 programming language (Rust), but **(todo)** targets WebAssembly to compensate
- knows how to split words into syllables
- accepts *y* before syllabic consonants in zi'evla (eg *sincyrboua*), except before *ml* and *mr* if that would
  cause a problem (eg *\*xlastmlu* isn't a zi'evla because its y-form *\*xlastymlu* is a slinku'i)
- does not separate the logic for CLL lujvo and zi'evla lujvo, as this causes many problems
- does as much as it can right-to-left to better reflect how Lojban morphology actually works
